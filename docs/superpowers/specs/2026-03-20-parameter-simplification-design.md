# parameter.py Simplification — Design Spec

**Date:** 2026-03-20
**Status:** Approved

---

## Context

`parameter.py` defines 6 classes (1 base + 5 concrete) representing search space parameter types used throughout f3dasm. The file is ~1,100 lines, most of which is boilerplate: each concrete class manually defines `__init__`, `__str__`, `__repr__`, `__eq__`, `_copy()`, and `to_dict()` with nearly identical structure. The goal is to cut this repetition by ~50% without changing any public API or behavior.

---

## Approach: `@dataclass` + registry

### 1. Fix `_type` ClassVar inconsistencies

Four classes have incorrect or misplaced `_type` values that prevent using `_type` as a reliable serialization key:

| Class | Current | Fix |
|---|---|---|
| `ConstantParameter` | inherits `"object"` from `Parameter` | `_type: ClassVar[str] = "constant"` |
| `CategoricalParameter` | explicitly `"object"` | `_type: ClassVar[str] = "category"` |
| `ArrayParameter` | inherits `"object"` | `_type: ClassVar[str] = "array"` |
| `DiscreteParameter` | `self._type = "int"` in `__init__` (line 717) | `_type: ClassVar[str] = "int"` at class level; **delete** the `self._type = "int"` instance assignment — it must not be carried into `__post_init__` |

### 2. Apply `@dataclass` to the 5 concrete subclasses

`Parameter` (base class) stays a regular class. Each concrete subclass becomes a dataclass:

- **`__init__`** replaced by field declarations; validation moves to `__post_init__`
- **`__repr__`** auto-generated by dataclass (uses `repr()` on each field value — identical output to current custom `__repr__`)
- **`__str__`** removed — defaults to `__repr__` (same content in all current concrete classes)
- **`__eq__`** auto-generated for `ConstantParameter`, `ContinuousParameter`, `DiscreteParameter`; kept custom (`@dataclass(eq=False)`) for `CategoricalParameter` (set comparison) and `ArrayParameter` (numpy)
- **`_copy()`** removed from all classes; the only call site (`Domain._copy()` lines 175–176) is replaced with `copy.copy(v)`

**Before** (`ContinuousParameter`, ~60 lines):
```python
def __init__(self, lower_bound=-inf, upper_bound=inf, log=False):
    super().__init__()
    self.lower_bound = float(lower_bound)
    self.upper_bound = float(upper_bound)
    self.log = log
    if self.log and self.lower_bound <= 0.0: raise ValueError(...)
    self._validate_range()

def __str__(self): return f"ContinuousParameter(lower_bound=..., ...)"
def __repr__(self): return f"ContinuousParameter(lower_bound=..., ...)"
def __eq__(self, __o): ...
def _copy(self): return ContinuousParameter(...)
def to_dict(self): ...  # calls super, overrides type key, adds 3 fields
```

**After** (`ContinuousParameter`, ~20 lines):
```python
@dataclass
class ContinuousParameter(Parameter):
    _type: ClassVar[str] = "float"
    lower_bound: float = field(default_factory=lambda: float("-inf"))
    upper_bound: float = field(default_factory=lambda: float("inf"))
    log: bool = False

    def __post_init__(self):
        super().__init__()
        self.lower_bound = float(self.lower_bound)
        self.upper_bound = float(self.upper_bound)
        if self.log and self.lower_bound <= 0.0:
            raise ValueError(...)
        self._validate_range()
```

**`CategoricalParameter` specifics:** Convert `categories` to `list` in `__post_init__` to normalise any `Iterable` input and ensure `len()` and serialisation work reliably:
```python
@dataclass(eq=False)
class CategoricalParameter(Parameter):
    _type: ClassVar[str] = "category"
    categories: list[Any] = field(default_factory=list)

    def __post_init__(self):
        super().__init__()
        self.categories = list(self.categories)
        self._check_duplicates()
```

**`ArrayParameter` specifics:** Use `@dataclass(eq=False)` and fix the pre-existing `__eq__` bug (comparing numpy arrays with `==` returns an array, not bool):
```python
def __eq__(self, other: Parameter) -> bool:
    if not isinstance(other, ArrayParameter):
        return False
    return (
        self.shape == other.shape
        and np.array_equal(self.lower_bound, other.lower_bound)
        and np.array_equal(self.upper_bound, other.upper_bound)
    )
```

### 3. Simplify `to_dict()` in base class using `dataclasses.fields()`

Replace the 5 per-class `to_dict()` overrides with a single implementation on `Parameter`:

```python
def to_dict(self) -> dict:
    d = {
        "type": self._type,
        "to_disk": self.to_disk,
        "store_function": pickle.dumps(self.store_function).hex() if self.store_function else None,
        "load_function": pickle.dumps(self.load_function).hex() if self.load_function else None,
    }
    if dataclasses.is_dataclass(self):
        for f in dataclasses.fields(self):
            d[f.name] = getattr(self, f.name)
    return d
```

This works because dataclass field names exactly match the dict keys used in the current per-class overrides. Note: `store_function` and `load_function` are NOT dataclass fields (they live on the base class), so they are handled explicitly above and not double-written by the loop.

**numpy serialisation note:** `ArrayParameter.to_dict()` stores `lower_bound` and `upper_bound` as `np.ndarray` objects — this is a pre-existing behaviour (unchanged by this refactor). JSON serialisation of `ArrayParameter` was already not supported; this is out of scope.

**`CategoricalParameter` serialisation change:** After the refactor, `categories` is always stored as a `list` in `to_dict()` output (because `__post_init__` normalises to `list`). Previously the raw iterable was stored unchanged. This is a safe, controlled improvement.

### 4. Replace `from_dict()` if/elif chain with a registry

Current: 60-line if/elif block manually constructing each type.

After:
```python
# Built after all classes are defined (end of module)
_PARAM_REGISTRY: dict[str, type[Parameter]] = {cls._type: cls for cls in PARAMETERS}

# In Parameter.from_dict():
@classmethod
def from_dict(cls, param_dict: dict) -> Parameter:
    param_type = param_dict["type"]
    store_function = pickle.loads(bytes.fromhex(param_dict["store_function"])) if param_dict.get("store_function") else None
    load_function = pickle.loads(bytes.fromhex(param_dict["load_function"])) if param_dict.get("load_function") else None

    if param_type == "object":
        return Parameter(to_disk=param_dict["to_disk"], store_function=store_function, load_function=load_function)

    param_cls = _PARAM_REGISTRY.get(param_type)
    if param_cls is None:
        raise ValueError(f"Unknown parameter type: {param_type}")
    fields = {f.name for f in dataclasses.fields(param_cls)}
    kwargs = {k: v for k, v in param_dict.items() if k in fields}
    return param_cls(**kwargs)
```

Note: `store_function`/`load_function` are not dataclass fields on any concrete subclass, so the field filter correctly excludes them — consistent with current behaviour where concrete subclasses don't accept those arguments.

## `copy.copy()` assumption

`copy.copy()` creates a shallow copy without running `__init__`/`__post_init__`, bypassing type coercion and validation. This is safe because all `Parameter` objects in a `Domain` are already valid and fully constructed. `Domain._copy()` is the only call site, and it copies parameters that were validated at construction time.
